# 보안 리뷰

코드/변경을 공격자 관점에서 점검한다. "이 입력을 신뢰할 수 있는가?"를 모든
경계에서 묻는다. 발견 시 심각도와 구체적 수정안을 함께 제시한다.

## 점검 항목 (OWASP 기반)
- **인젝션**: SQL/NoSQL/OS 명령/LDAP. 문자열 연결 대신 파라미터 바인딩·
  허용목록. 셸 호출 시 사용자 입력 차단.
- **인증/세션**: 약한 비밀번호 정책, 토큰 만료·회전 부재, 세션 고정,
  안전하지 않은 쿠키 플래그(HttpOnly/Secure/SameSite).
- **접근 제어**: 객체 수준 권한 검사 누락(IDOR), 수직/수평 권한 상승,
  기본 허용 정책.
- **민감정보**: 하드코딩된 시크릿/키, 로그·에러에 노출된 PII, 평문 저장,
  약한 암호화.
- **입력 검증**: 경로 순회(`../`), 안전하지 않은 역직렬화, SSRF, 무제한 업로드.
- **의존성**: 알려진 취약 버전, 핀 고정 부재. (`pip audit`, `npm audit` 등 제안)
- **설정**: 디버그 모드 노출, 과도한 CORS, 비밀이 담긴 환경/설정 파일 커밋.

## 보고
- `[심각도] 위치 — 취약점 유형`: 공격 시나리오 한 줄 + 수정안.
- 심각도: `critical` / `high` / `medium` / `low`.
- 추측성 경고를 남발하지 않는다. 실제로 악용 가능한 경로를 우선한다.
- 시크릿을 발견해도 값 전체를 출력하지 말고 위치만 지목한다.
