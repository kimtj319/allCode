# 67. 에이전트 진척도 평가 (vs Codex / Claude Code) — 2026-06-14

이번 세션의 실 PTY/TTY 검증과 코드 점검 결과에 기반한 추정치. **하니스(에이전트
엔지니어링)** 와 **하부 모델 역량(wise-lloa-max vs gpt-5.5/Claude)** 을 분리해 평가한다.
하니스는 Codex/Claude Code에 근접했고, 남은 격차의 상당 부분은 모델 역량.

## 종합 추정

- **하니스(에이전트 엔지니어링) 성숙도**: **약 85–88%** (Codex/Claude Code 대비).
  라우팅·도구·컨텍스트·생성/수정/분석 파이프라인·UI/UX·안전 격리가 대부분 동등 수준.
- **종단 사용자 체감(품질 포함)**: **약 78–82%**. 위 하니스에 하부 모델 역량(코드 정확도·
  추론 깊이·도구-following 신뢰성) 격차가 곱해진 결과. wise-lloa-max < gpt-5.5/Claude.

### 격차의 성격
- **하니스로 메울 수 있는 잔여(~12–15%)**: 슬래시 명령 확충(/diff·/compact·@멘션), 전체-transcript
  기반 컨텍스트 옵션, OS레벨 샌드박스, 웹검색 백엔드 구성.
- **모델 의존(하니스 밖)**: 크로스커팅 수정의 edit emission 신뢰성, 분석 본문 심층도, 복잡
  구현 정확도. 더 강한 모델을 붙이면 종단 체감이 하니스 성숙도(≈85%)에 수렴.

## 결론 한 줄
**에이전트(하니스) 진척도는 Codex/Claude Code 대비 약 85%**, 동일 모델 가정 시 종단 품질
약 80%; 남은 격차는 절반은 하니스 기능 확충, 절반은 하부 모델 역량.
