# 66. 멀티턴 컨텍스트 메모리 검증 + 보강 (2026-06-14)

## 발견한 격차
교차턴 메모리(`ContextBuilder`)는 턴마다 다음을 저장·주입했음:
- 사용자 측: `extract_session_note`로 **명시적 별칭("앞으로 X는 Y")만** 추출 → 일반 질문은
  `None`이라 저장 안 됨.
- assistant 측: 답변 요약(≤1200자) 최근 5개.

→ 모델이 다음 턴에서 "내가 답한 것"은 알아도 **"사용자가 무엇을 물었는지"는 모름**.
사람식 후속질문("그 두 번째 방법", "내가 처음에 뭐 물어봤지?")의 사용자측 맥락이 누락.

## 보강
- `ContextBuilder.remember_user_prompt(session_id, prompt)` 추가: 매 턴의 사용자 프롬프트를
  압축(≤240자)해 세션별 최근 8개 보관(`_recent_prompts`).
- `runtime.run_agent_turn`이 턴 완료마다 `remember_user_prompt` 호출(헤드리스·TUI 공용 경로).
- `session_note_sections`가 최근 사용자 프롬프트와 assistant 요약을 **교대로(오래된 순)**
  "Recent conversation turns" 블록으로 주입 → 모델이 실제 대화 흐름(질문↔답변)을 봄.
- TUI 러너는 context_builder를 턴 간 공유하므로 인터랙티브 멀티턴에서 in-memory로 누적됨.

1. "반복 처리 효율적 방법 3가지" → 제너레이터/청크/병렬 제시.
2. "그 중 두 번째 방법 자세히" → **청크 단위 읽기**를 정확히 심층 설명(연속성).
3. "방금 방법들 중 메모리 부족 대용량에 가장 적합한 건?" → **청크 단위 읽기**(이전 맥락 참조).

   처리할 때 효율적인 방법 3가지를 간단히 알려줘"** 그대로 회상. ← 보강 전 불가능, 보강 후 성공.
