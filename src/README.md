# src 폴더 코드 요약

`src/allCode` 디렉터리는 전체 프로젝트의 핵심 로직을 여러 패키지로 나누어 구성하고 있습니다. 각 패키지는 다음과 같은 역할을 담당합니다.

## core
- **역할**: Pydantic 기반의 핵심 데이터 모델, 이벤트 시스템, 결과와 증거(evidence) 모델, 에러 정의, 공통 경로 패턴 등을 정의합니다.
- **주요 파일**: `model.py`, `event.py`, `result.py` 등.

## llm
- **역할**: LLM(대형 언어 모델)과의 통신을 추상화한 프로토콜을 제공하고, OpenAI 호환 어댑터와 테스트용 Fake LLM 구현을 포함합니다.
- **주요 파일**: `client.py`, `adapters/openai.py`, `fake.py` 등.

## agent
- **역할**: 사용자 의도 추출, 라우팅, 정책 결정, 프롬프트 생성, 에이전트 루프와 회복 메커니즘, 컨텍스트 빌더, 최종 응답 생성 등을 담당합니다.
- **주요 파일**: `router.py`, `policy.py`, `loop.py`, `context_builder.py` 등.

## tools
- **역할**: 도구 계약 정의, 도구 레지스트리, 실행기, 승인 로직, 파일/검색/쉘/웹 등 내장 도구들을 구현합니다.
- **주요 파일**: `base.py`, `registry.py`, `executor.py`, `approval.py` 등.

## workspace
- **역할**: 워크스페이스 루트 관리, 안전한 경로 해석, 파일 인덱싱 및 심볼 추출을 담당합니다.
- **주요 파일**: `workspace.py`, `indexer.py` 등.

## memory
- **역할**: 계층적 메모리 구조, 세션 요약, 최근 타깃 관리, 레포지토리 맵, 자동 메모리 인박스 등을 제공하여 컨텍스트를 축소·보관합니다.
- **주요 파일**: `store.py`, `session.py`, `selector.py` 등.

## generation
- **역할**: 다양한 언어(파이썬, Node, Go 등)용 프로젝트 생성 전략을 레지스트리 형태로 제공하고, 코드 생성 워크플로우를 조정합니다.
- **주요 파일**: `strategies/*.py` 등.

## tui
- **역할**: 기본 터미널‑네이티브 UI와 선택적 Textual UI를 구현합니다. 입력 컴포저, 히스토리, 스피너, 마크다운 렌더링 등 UI 전반을 담당합니다.
- **주요 파일**: `ui.py`, `textual_app.py` 등.

## config
- **역할**: 설정 스키마 정의, 기본값 제공, 환경·파일·CLI 플래그 등 다양한 소스에서 설정을 병합하는 로더를 구현합니다.
- **주요 파일**: `schema.py`, `loader.py` 등.

## 기타 파일
- `__init__.py`: 패키지 초기화와 버전 정의.
- `main.py`, `headless.py`, `runtime.py`: CLI 엔트리포인트와 런타임 로직.
- `__main__.py`: `python -m allCode` 실행 시 진입점.

이와 같이 `src/allCode`는 **핵심 로직, LLM 연동, 에이전트 흐름, 도구 시스템, 워크스페이스 관리, 메모리, 코드 생성, UI, 설정** 등 프로젝트 전반을 모듈화하여 유지보수와 확장을 용이하게 설계되었습니다.
